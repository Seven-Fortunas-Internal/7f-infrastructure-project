### 4. Public/Private Mirroring Strategy

**Discovery:** GitHub Private Mirrors App (PMA) enables this pattern

**How It Works:**
1. Develop privately in internal org
2. Review & approve internally (security, legal, compliance)
3. PMA automatically syncs approved branches to public fork
4. Open contributions to public repo with full history intact
5. No commit rewriting - preserves attribution

**Use Cases for Seven Fortunas:**
- CISO Assistant compliance framework (private controls, public methodology)
- Open-source projects with internal development phase
- Documentation (internal detailed, public sanitized)
- Thought leadership (research done privately, published strategically)

### GitHub Organization Best Practices

**Key Findings:**
- **Minimize number of organizations** (promotes collaboration, innersourcing)
- Single org for startups where everyone collaborates
- At least 2 people with owner role (continuity)
- Enable security features by default: Dependabot, secret scanning, dependency graph
- Teams provide organizational hierarchy without org proliferation

**Sources:**
- [GitHub Organization Best Practices](https://docs.github.com/en/enterprise-cloud@latest/admin/concepts/best-practices)
