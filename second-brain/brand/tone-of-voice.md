# Tone of Voice

## Brand Voice Personality

Seven Fortunas communicates with a voice that is:

- **Professional yet approachable**: We're experts, but we don't hide behind jargon
- **Clear and direct**: We respect the reader's time with concise, purposeful language
- **Optimistic and empowering**: We believe in what we're building and inspire that belief
- **Technically grounded**: We explain complex concepts with precision and clarity
- **Inclusive**: We welcome all audiences and explain concepts at multiple levels

## Writing Style Guidelines

### Tone Rules

**DO:**
- Use active voice ("We deploy features" not "Features are deployed")
- Write in second person when addressing users ("You can configure...")
- Use contractions naturally ("we're", "don't", "it's")
- Break up complex sentences into simpler ones
- Use parallel structure in lists

**DON'T:**
- Use passive voice or overly formal language
- Assume excessive technical knowledge
- Write in all caps (EXCEPT for acronyms when appropriate)
- Use exclamation points excessively
- Use corporate jargon or empty marketing speak

### Examples

**DON't:** "Deployment configurations must be established through the utilization of the configuration management interface."
**Do:** "Configure your deployments in the settings panel."

**Don't:** "Our revolutionary AI-native infrastructure empowers unprecedented digital transformation!!!"
**Do:** "Build faster with AI-native infrastructure that adapts to your needs."

## Terminology Glossary

| Term | Preferred Usage | Context |
|------|-----------------|---------|
| User | "User", "team member" | When referring to people |
| Feature | Specific capability | Avoid generic "thing" or "functionality" |
| Dashboard | Dashboard (not "panel", "interface") | Refers to monitoring/analytics views |
| Configure | Setup, configure (not "instantiate") | Describing user actions |
| Deploy | Deploy, launch (context-dependent) | Releasing to production |
| API | API (spell out on first use: "Application Programming Interface") | Technical integrations |

## Messaging Templates

### Success Messages
"✓ [Action] completed successfully. [Optional next step or context]."

Example: "✓ Deployment started. View progress in the activity log."

### Error Messages
"[Issue description]. [How to fix or where to find help]."

Example: "Configuration validation failed. Check the syntax in your config file or contact support."

### Confirmations
"[Clear action description]. This [cannot be reversed / can be undone]."

Example: "Delete this API key? This action cannot be reversed."

## Social Media Voice

- **Twitter/X**: Concise, engaging, conversational. Use emoji sparingly.
- **LinkedIn**: Professional but not corporate-speak. Share insights and company updates.
- **GitHub**: Technical, clear, focused. Follow developer conventions.
- **Blog**: Informative and engaging. Tell stories, not just announcements.

## Email Communication

**Subject Lines:**
- Be specific: "New feature: Custom dashboards now available"
- Avoid clickbait: "You won't believe what we shipped" ❌

**Opening:**
- Address reader directly or use team name
- Lead with value or key news

**Closing:**
- Invite questions or feedback
- Clear next steps

**Example:**
```
Hi Team,

We're excited to announce custom dashboards are now available.
This lets you build monitoring views tailored to your needs.

Get started: [link]

Questions? Reply to this email.

Best,
Seven Fortunas
```
