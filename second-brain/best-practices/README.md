# Best Practices

This domain contains engineering standards, security protocols, and operational best practices for Seven Fortunas.

## Navigation

- **coding-standards.md** — Code style, patterns, and quality standards
- **security.md** — Security practices, threat models, and compliance
- **git-workflow.md** — Git branching strategy, commit standards, and code review process

## Purpose

The Best Practices domain establishes consistent standards and workflows across technical and operational teams, ensuring code quality, security, and efficient collaboration.

## Key Documents

### coding-standards.md
Code quality and standards:
- Language-specific style guides (Python, JavaScript, Go, etc.)
- Design patterns and architectural conventions
- Testing standards and coverage requirements
- Documentation requirements and API conventions
- Performance and optimization guidelines
- Refactoring and technical debt management

### security.md
Security and compliance:
- Security principles and threat modeling
- Authentication and authorization standards
- Data protection and encryption practices
- Secret management and credential handling
- Vulnerability disclosure and incident response
- Compliance frameworks (SOC 2, GDPR, etc.)
- Security review and audit processes

### git-workflow.md
Version control and collaboration:
- Branching strategy (main, develop, feature branches)
- Commit message conventions and standards
- Pull request process and code review expectations
- Merge strategies and conflict resolution
- Release and tagging conventions
- CI/CD integration points
