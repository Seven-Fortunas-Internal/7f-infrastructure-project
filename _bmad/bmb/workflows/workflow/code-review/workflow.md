# Code Review

**Goal:** Ensure code quality through systematic architectural and implementation review.

## Steps

1. **Load Code Changes**: Review pull request or commit set
2. **Understand Context**: Read description, architecture, related issues
3. **Review Architecture**: Assess design patterns and structure
4. **Check Implementation**: Verify correctness, efficiency, maintainability
5. **Test Locally**: Check out code and run locally if applicable
6. **Provide Feedback**: Document comments with specific examples
7. **Suggest Improvements**: Recommend refactoring, optimization, or pattern changes
8. **Approve or Request Changes**: Document decision with rationale

## Tools

- Code review platforms (GitHub, GitLab, etc.)
- Static analysis tools (linters, SonarQube, etc.)
- IDE plugins for inline review
- Diff/merge viewers
- Architecture visualization tools
